# DFS (깊이 우선 탐색)

## 1. 언제 떠올려야 하는가?
- 경로의 특징을 저장해야 할 때 (경로 자체 기록)
- **백트래킹**: 모든 가능한 경우의 수(순열/조합)를 탐색할 때
- 사이클 탐지

## 2. 사고 흐름 / 템플릿
1. **종료 조건(Base Case)** 설정: 재귀 탈출 조건
2. **방문 처리**: 현재 노드 방문 (`visited[node] = true`)
3. **인접 탐색**: 연결된 노드 중 방문하지 않은 곳으로 재귀 호출 (`dfs(next_node)`)
4. **(백트래킹 시) 상태 복구**: 재귀에서 돌아오면 `visited`를 다시 `false`로

## 3. 자주 하는 실수
- 종료 조건을 제대로 설정하지 않아 무한 재귀 (Stack Overflow)
- 백트래킹 시 상태 복구(`visited = false`)를 누락함
- 재귀 깊이가 깊어질 경우(`sys.setrecursionlimit` 등 언어별 한계 확인 필요)
