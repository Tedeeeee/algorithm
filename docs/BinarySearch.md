# Binary Search (이진 탐색)

## 1. 언제 떠올려야 하는가?
- 입력 범위가 매우 클 때 (N > 1,000,000)
- "최댓값/최솟값을 찾아라" -> 결정 문제(Parametric Search)로 변환 가능할 때
- 데이터가 정렬되어 있거나, 정렬해서 풀 수 있을 때

## 2. 사고 흐름 / 템플릿
1. **탐색 범위 설정**: `low`, `high` 초기값 결정
2. **반복문**: `while (low <= high)`
3. **중간값**: `mid = (low + high) / 2` (오버플로우 주의: `low + (high - low) / 2`)
4. **조건 확인**: 
   - `check(mid)`가 `Target`과 일치하면 `return`
   - 작으면 `low = mid + 1`
   - 크면 `high = mid - 1`

## 3. 자주 하는 실수
- **범위 설정**: `high`를 N으로 할지 N-1로 할지 (인덱스 vs 값)
- **무한 루프**: `low`와 `high` 갱신 시 `+1`, `-1`을 빼먹음
- **Upper/Lower Bound**: 중복 원소가 있을 때 어느 쪽을 찾아야 하는지 혼동
