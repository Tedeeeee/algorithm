# 🔵 배열 (Array) / 리스트 (List)

배열(Array)과 리스트(List)는  
**모든 알고리즘 문제의 출발점이 되는 기본 자료구조**다.

대부분의 문제에서 입력 데이터는  
👉 결국 **배열 형태로 저장되고 처리**된다.

코딩테스트에서 배열 문제는  
“자료구조 문제”라기보다는  
👉 **인덱스 관리 + 반복문 설계 문제**에 가깝다.

---

## 언제 배열 / 리스트를 떠올려야 하나?
아래 상황이면 거의 100% 배열부터 시작한다.

- 입력이 여러 개 주어진다
- 순서가 중요하다
- 인덱스로 접근해야 한다
- 연속된 데이터다
- 단순 비교 / 누적 / 카운트가 필요하다

📌 핵심 신호
> **“일단 배열에 담아라”**

---

## 배열 vs 리스트 (감각적으로 구분)

### Array
- 크기 고정
- 인덱스 접근 O(1)
- 메모리 구조가 단순

📌 언제 쓰나?
- 크기를 미리 알 수 있을 때
- 성능이 중요한 문제

---

### List (ArrayList)
- 크기 가변
- 인덱스 접근 O(1)
- 내부적으로 배열 사용

📌 언제 쓰나?
- 크기가 유동적일 때
- 삽입/삭제가 잦을 때

---

## 배열 문제의 핵심 사고
배열 문제는 알고리즘보다  
**“인덱스를 어떻게 굴릴지”**가 전부다.

문제를 읽으면 바로 이걸 생각해야 한다.

- 0부터 시작인가? 1부터 시작인가?
- 배열 크기는 정확히 얼마인가?
- 추가 공간이 필요한가?
- 중첩 반복문이 가능한가?

📌 배열 문제의 난이도는  
👉 **로직이 아니라 실수 방지 능력**이다.

---

## 자주 나오는 배열 활용 패턴

### 1️⃣ 카운팅 배열
- 빈도 계산
- 숫자 / 문자 개수 세기

📌 주의
- 범위 초과
- 초기화 위치

---

### 2️⃣ 누적합 (Prefix Sum)
- 구간 합 빠르게 계산
- 반복 계산 제거

📌 주의
- 인덱스 off-by-one
- 누적합 배열 크기

---

### 3️⃣ 단순 탐색
- 최댓값 / 최솟값
- 조건 만족 여부

📌 주의
- 초기값 설정
- 비교 조건 방향

---

## 배열 문제에서 반드시 점검할 것
문제 풀기 전에 이 체크리스트를 훑어라.

- 배열 크기 충분한가?
- 인덱스 범위 안전한가?
- 반복문 종료 조건 정확한가?
- 입력 개수와 반복 횟수가 일치하는가?
- 시간복잡도 괜찮은가?

---

## 자주 하는 실수 (진짜 많이 나옴)
- 배열 크기 부족
- 인덱스 하나 밀림
- 초기화 누락
- for문 범위 실수
- 쓸데없는 배열 복사

📌 배열 문제는  
👉 **사소한 실수 하나로 바로 오답**이다.
