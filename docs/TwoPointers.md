# Two Pointers (투 포인터)

## 1. 언제 떠올려야 하는가?
- 1차원 배열에서의 구간 합, 부분 수열 문제
- O(N^2)으로 풀면 시간 초과가 나는 경우 (N > 10,000)
- 연속된 구간의 특성을 구할 때

## 2. 사고 흐름 / 템플릿
1. `start`, `end` 두 포인터를 0(또는 양 끝)에 둔다.
2. 현재 구간(`start` ~ `end`)의 상태(합, 개수 등)를 확인한다.
3. 조건에 따라:
   - 값이 부족하면 `end` 증가
   - 값이 넘치면 `start` 증가
   - 정답이면 기록 후 `start` 증가 (다음 해 탐색)

## 3. 자주 하는 실수
- `end`가 배열 범위를 벗어날 때의 처리를 누락함
- `start`가 `end`를 앞지르는 경우 (길이가 0이 되는 경우) 고려 미흡
- 초기값(0 vs 1) 및 구간 정의(닫힌 구간 vs 열린 구간) 혼동
