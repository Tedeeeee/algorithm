# 🧺 HashSet

HashSet은  
**중복을 허용하지 않고, 빠르게 존재 여부를 확인하는 자료구조**다.

코딩테스트에서 HashSet은  
👉 “자료를 저장한다”가 아니라  
👉 **‘이미 나왔는지’를 즉시 판단하기 위한 도구**다.

---

## 언제 HashSet을 떠올려야 하나?
문제에서 아래 신호가 보이면 HashSet을 의심한다.

- 중복 제거
- 이미 등장했는지 확인
- 방문 여부 관리
- 유일한 값만 유지
- 순서는 중요하지 않다

📌 핵심 신호
> **“있다 / 없다 만 알면 된다”**

---

## HashSet의 핵심 사고
HashSet의 본질은  
**“값 자체를 인덱스로 사용한다”**는 생각이다.

그래서 항상 이 질문을 던져야 한다.

> “이 값이  
> 이미 나왔는지만 알면 되는가?”

YES → HashSet  
NO → HashMap / 배열

---

## HashSet 사고 흐름 (공식처럼)

1. **중복을 허용하면 안 되는가?**
2. **존재 여부만 필요하지 않은가?**
3. **순서가 전혀 중요하지 않은가?**

📌 이 세 개가 전부 YES면  
👉 HashSet이 정답이다.

---

## HashSet의 대표 사용 패턴

### 1️⃣ 중복 제거
- 배열 / 리스트에서 중복 제거
- 유일한 값 집합 만들기

📌 출력 순서가 필요하면 정렬 추가

---

### 2️⃣ 방문 체크
- 그래프 / DFS / BFS
- 이미 방문한 노드 관리

📌 boolean 배열로 대체 가능한지도 고민

---

### 3️⃣ 빠른 포함 여부 검사
- 어떤 값이 있는지 확인
- 조건 만족 여부 판단

📌 contains()가 핵심

---

## HashSet 문제에서 반드시 점검할 것들
문제 풀기 전에 이걸 체크해라.

- 순서가 정말 필요 없는가?
- 중복 제거만 하면 되는가?
- 배열 / boolean 배열로 충분하지 않은가?
- HashMap이 더 적합하지 않은가?

---

## HashSet에서 자주 하는 실수
(실전에서 진짜 많이 나온다)

- 순서를 기대함
- 출력 순서 조건 무시
- 불필요하게 HashMap 사용
- equals / hashCode 고려 안 함 (객체)
- 초기화 누락

📌 HashSet은  
👉 **단순한 만큼 오해도 많다.**

---

## HashSet vs 다른 자료구조
이 구분이 되면 선택이 빨라진다.

- 중복 ❌ / 존재 여부 ⭕ → HashSet
- 개수 / 정보 필요 ⭕ → HashMap
- 범위 작음 → boolean 배열

---

## 시간복잡도 감각
HashSet 평균 시간복잡도:

- 삽입: O(1)
- 조회: O(1)
- 삭제: O(1)

📌 코테에서는  
👉 성능 걱정 없이 써도 된다.
