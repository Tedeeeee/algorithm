# Tree (트리 알고리즘)

## 1. 언제 떠올려야 하는가?
- 계층적(Hierarchical) 데이터 구조
- 부모-자식 관계가 명확할 때
- **트리 순회(Traversal)**: 전위, 중위, 후위
- LCA(최소 공통 조상) 문제

## 2. 사고 흐름 / 템플릿
- 대부분 **재귀(Recursion)**를 사용
1. **Base Case**: `node == null`이면 리턴
2. **Recursive Step**: 왼쪽 자식 방문 -> (처리) -> 오른쪽 자식 방문 (순서에 따라 순회 방식 결정)
3. **트리 DP**: 서브트리의 결과를 기반으로 현재 노드의 값을 결정

## 3. 자주 하는 실수
- 방향성 혼동 (부모 -> 자식 단방향인지, 양방향인지) - 양방향이면 `visited` 필요할 수 있음
- 루트 노드가 1번이라고 단정 짓는 실수 (문제 조건 확인)
- 이진 트리가 아닌 경우(N-ary tree)의 인접 리스트 처리
